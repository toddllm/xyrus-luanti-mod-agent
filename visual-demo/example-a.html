Example A: Cache Hit Scenario

The Fastest Path

When a player requests a node that's in a MapBlock already loaded in memory, Luanti can retrieve it almost instantly. This is the most common case during normal gameplay, as nearby blocks are kept in the cache.

Starting position: Player at (1234, 56, -789) looking at a stone node
Block position: (77, 3, -50)
Cache status: Block already loaded

1

Player Requests Node

The game engine needs to know what node the player is looking at or interacting with.

// src/client/game.cpp void Game::handlePointingAtNode(const PointedThing &pointed) { v3s16 nodepos = pointed.node_undersurface; // (1234, 56, -789) MapNode n = map->getNode(nodepos); }
2

Calculate Block Position

Convert world coordinates to block coordinates by dividing by 16 (block size).

// src/map.cpp v3s16 getNodeBlockPos(v3s16 p) { return v3s16( floor(p.X / 16), // 1234 / 16 = 77.125 → 77 floor(p.Y / 16), // 56 / 16 = 3.5 → 3 floor(p.Z / 16) // -789 / 16 = -49.3 → -50 ); }
3

Check Block Cache

Look up the block in the memory cache. In this scenario, it's a hit!

// src/map.cpp MapBlock *Map::getBlockNoCreateNoEx(v3s16 p) { // First check cache - FAST! MapBlock *block = m_blocks_cache.lookup(p); if (block) return block; // Cache hit! No database access needed // (Database load path not taken in this scenario) }
4

Calculate Node Index

Calculate the position within the 16×16×16 block and convert to array index.

// Calculate relative position within block v3s16 relpos = p - blockpos * 16; // relpos = (1234, 56, -789) - (1232, 48, -800) = (2, 8, 11) // Convert to array index (Z-Y-X order) u32 index = relpos.Z * 256 + relpos.Y * 16 + relpos.X; // index = 11 * 256 + 8 * 16 + 2 = 2816 + 128 + 2 = 2946
5

Return Node Data

Access the node directly from the block's array and return to the game engine.

// src/mapblock.h MapNode getNodeNoCheck(s16 x, s16 y, s16 z) { return data[z * 256 + y * 16 + x]; } // Result: MapNode with content_id = 1 (stone in this block's mapping)
